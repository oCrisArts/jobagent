# 🧪 Estratégia de Testes - Validação

## 🎯 Objetivo
Aprender observando usuários reais e métricas de engajamento

## 👥 Teste com Usuários (Maze)

### Cenário 1: Onboarding Completo
**Participantes**: 5 Lucases (Dev Jr)
**Duração**: 15 minutos
**Tasks**:
1. "Faça signup com Google"
2. "Upload seu CV"
3. "Procure vagas para Frontend Junior"
4. "Adapte seu currículo para a primeira vaga"
5. "Veja seu SSI Score"

**Métricas**:
- Task completion: >80%
- Time per task: <3 min cada
- Clicks para signup: <5
- Confusão em upload? (Maze heatmap)

### Cenário 2: Upgrade Trigger
**Participantes**: 3 Lucas que completaram S1
**Task**:
1. "Tente adaptar CV novamente"
2. "Clique em fazer upgrade"
3. "Complete checkout no Stripe"

**Métricas**:
- Vê a mensagem "Sua adaptação foi usada"?
- Clica em "Upgrade"?
- Completa Stripe?
- Sem fricção?

## 📊 HEART Framework

### Happiness
```
NPS: "Recomendaria a 0-10?"
Target: 7+

Satisfaction: "Resolveu seu problema?"
Target: 4/5+
```

### Engagement
```
DAU: Users ativos por dia
Target: 20% of signups

Frequency: Vezes por semana
Target: 2-3x (mission-driven)

Job Search: Buscas por semana
Target: 5+
```

### Adoption
```
Signup → First Adaptation: %
Target: >70%

First Adaptation → Upgrade: %
Target: 5%+
```

### Retention
```
D1: 70% (primeira semana)
D7: 40% (segunda semana)
D30: 20% (mês)

Target: Melhorar com daily missions
```

### Task Success
```
Onboarding: >80% completion
Adaptation: >95% success
Upgrade: >90% de quem vê paywall
```
